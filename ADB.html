Android Debug Bridge (adb) is a versatile command line tool that lets you communicate with an emulator instance or connected Android-powered device.ADB 是一种通用的命令行工具。 开发者可以用来与模拟器实例或安卓设备通信
 It is a client-server program that includes three components:
它是客户-服务端程序， 有三个组成部分：
  • A client, which runs on your development machine. You can invoke a client from a shell by issuing an adb command. Other Android tools such as DDMS also create adb clients.
  • 客户端， 运行在开发机器上。 可以发出 adb命令 在adb shell中调用客户端。 其他的安卓工具像DDMS，也能创建adb客户端
  • A server, which runs as a background process on your development machine. The server manages communication between the client and the adb daemon running on an emulator or device.
  • 服务器， 在开发机器上以后后台进程运行。这个服务器用来管理客户端与运行在模拟器或设备上的守护进程之间的通信
  • A daemon, which runs as a background process on each emulator or device instance.
  • 守护进程， 作为后台进程运行在模拟器或设备实例上

You can find the adb tool in <sdk>/platform-tools/. adb工具的路径

When you start an adb client, the client first checks whether there is an adb server process already running. If there isn't, it starts the server process. When the server starts, it binds to local TCP port 5037 and listens for commands sent from adb clients—all adb clients use port 5037 to communicate with the adb server.
The server then sets up connections to all running emulator/device instances. It locates emulator/device instances by scanning odd-numbered ports in the range 5555 to 5585, the range used by emulators/devices. Where the server finds an adb daemon, it sets up a connection to that port. Note that each emulator/device instance acquires a pair of sequential ports — an even-numbered port for console connections and an odd-numbered port for adb connections.
adb客户端启动后， 会先检查adb 服务进程是否已经运行。 如果没有， 会先启动服务进程， 这个服务启动后会绑定TCP协议中的本地端口 :5037， 并且通过这个端口号与客户端通信， 监听所有来自客户端的命令。
模拟器/客户端实例会使用5555-5585范围内的奇数端口号， 服务器启动后会扫描这些端口来定位实例， 一一建立连接。 需要注意的是，每一个实例都需要一对连续的端口号， 偶数端口号用于与控制台连接， 奇数端口号用于与adb server连接
 For example:

Emulator 1, console: 5554
Emulator 1, adb: 5555
Emulator 2, console: 5556
Emulator 2, adb: 5557
and so on...

As shown, the emulator instance connected to adb on port 5555 is the same as the instance whose console listens on port 5554.
如上所示， 由5555端口连接到adb 的与由5554连接到控制台的是同一个实例

Once the server has set up connections to all emulator instances, you can use adb commands to access those instances. Because the server manages connections to emulator/device instances and handles commands from multiple adb clients, you can control any emulator/device instance from any client (or from a script).
一旦服务器建立了所有模拟器实例的连接， 你就可以用adb命令来访问那些实例。服务管理多个模拟器/设备实例的连接和处理来自多个adb客户端的命令，开发可以控制来自任何一个客户端的任何一个实例


Enabling adb Debugging 启用adb调试
In order to use adb with a device connected over USB, you must enable USB debugging in the device system settings, under Developer options.
要使设备与adb通过USB连接， 须启动设备上系统设置里 开发者选项中的USB 调试
On Android 4.2 and higher, the Developer options screen is hidden by default. To make it visible, go to Settings > About phone and tap Build number seven times. Return to the previous screen to find Developer options at the bottom.
在Android 4.2或以上的设备中， 开发者选项是默认以藏的。 打开设置>关于手机， 找到版本号并连续点击7次， 便能打开开发者选项。
On some devices, the Developer options screen may be located or named differently.
有些设备中开发者选项的页面或者命名不同

Note: When you connect a device running Android 4.2.2 or higher to your computer, the system shows a dialog asking whether to accept an RSA key that allows debugging through this computer. This security mechanism protects user devices because it ensures that USB debugging and other adb commands cannot be executed unless you're able to unlock the device and acknowledge the dialog. This requires that you have adb version 1.0.31 (available with SDK Platform-tools r16.0.1 and higher) in order to debug on a device running Android 4.2.2 or higher.
注意： 当Android 4.2.2或以上版本的设备连接到电脑的时候， 系统会弹出一个对话框，询问你是否接受RSA Key允许通过这台电脑调试。 这种安全机制是用来保护设备的，用户只有在接受这个对话框或者解锁设备的情况下才能调试获执行其他的adb命令 。要想在这类型的设备上做调试， 你得需要adb 1.0.31（SDK Platform-tools r16.0.1 获更高版本以上会有）   
For more information about connecting to a device over USB, read Using Hardware Devices.
更多有关USB连接设备的信息， 请阅读此链接

Syntax语法

You can issue adb commands from a command line on your development machine or from a script. The usage is:
开发者可以在开发机器或者脚本中发出一条adb命令。 具体使用方法如下
adb [-d|-e|-s <serialNumber>] <command>
If there's only one emulator running or only one device connected, the adb command is sent to that device by default. If multiple emulators are running and/or multiple devices are attached, you need to use the -d, -e, or -s option to specify the target device to which the command should be directed.
如果只连接了一台设备或模拟器， adb命令会默认发送给该设备。 如果有堕胎模拟器获设备的， 则需使用-d, -e, or -s option 选项来指定命令发出的目标设备

Querying for Emulator/Device Instances 查询实例
Before issuing adb commands, it is helpful to know what emulator/device instances are connected to the adb server. You can generate a list of attached emulators/devices using the devices command:在发出adb命令之前便知道adb server中连接了哪些实例将很有帮助。 你可以使用以下命令先列出连接的实例的清单
adb devices
In response, adb prints this status information for each instance:
adb响应后将会打印出每个实例的下列状态信息
  • Serial number — A string created by adb to uniquely identify an emulator/device instance by its console port number. The format of the serial number is <type>-<consolePort>. Here's an example serial number: emulator-5554
    序列号： adb将实例连接的控制台端口号生成的字符串。 序列号的格式是     <type>-<consolePort>
  • State — The connection state of the instance may be one of the following:状态， 实例连接的状态有以下3种
    • offline — the instance is not connected to adb or is not responding. 离线-实例没有连接到adb中或者没有响应
    • device — the instance is now connected to the adb server. Note that this state does not imply that the Android system is fully booted and operational, since the instance connects to adb while the system is still booting. However, after boot-up, this is the normal operational state of an emulator/device instance.
      设备 - 实例已经练到服务。 请注意这种状态并不说明系统已经完全boot并且可以操作。因为实例连接的时候系统仍然在booting。 不过， 在系统启动后， 这将是实例正常可操作的状态
    • no device — there is no emulator/device connected. 无设备- 没有实例连接
The output for each instance is formatted like this: 每个实例的输出结果的格式如下所示
[serialNumber] [state]

Here's an example showing the devices command and its output:

adb devices
List of devices attached
emulator-5554  device
emulator-5556  device
emulator-5558  device

Directing Commands to a Specific Emulator/Device Instance 指向命令到特定的实例
If multiple emulator/device instances are running, you must specify a target instance when issuing adb commands. To do so, use the -s option in the commands. The usage for the -s option is:
如果有多台实例在运行， 那么当发出指令时你必须得指定目标实例。 可以使用命令中的 -s 选项
adb -s <serialNumber> <command>

As shown, you specify the target instance for a command using its adb-assigned serial number. You can use the devicescommand to obtain the serial numbers of running emulator/device instances. For example:

adb -s emulator-5556 install helloWorld.apk

Note that, if you issue a command without specifying a target emulator/device instance while multiple devices are available, adb generates an error.

If you have multiple devices available (hardware or emulated), but only one is an emulator, simply use the -e option to send commands to the emulator. Likewise if there's multiple devices but only one hardware device attached, use the -doption to send commands to the hardware device.
注意， 当有多台可用设备连接， 你发出命令去人没有指定目标实例的时候， adb将会报错。
如果你有多台可用设备， 但只有一个模拟器， 那只需要用-e 选项将命令发送到模拟器上。 类似的有多台设备，却只有一个应将设备的时候， 你可以使用 -d 向硬件发出命令

Installing an Application 安装应用
You can use adb to copy an application from your development computer and install it on an emulator/device instance. To do so, use the install command. With the command, you must specify the path to the .apk file that you want to install:
你可以用 adb将应用从开发机器中来复制应用并将其安装到实例中。 可以使用install命令， 但必须要指定你要安装的那个.apk 文件的路径。
adb install <path_to_apk>
For more information about how to create an .apk file that you can install on an emulator/device instance, see Building and Running
更多有关生成.apk 文件的信息 请看 Building and Running
Note that, if you are using Android Studio, you do not need to use adb (or aapt) directly to install your application on the emulator/device. Instead, Android Studio handles the packaging and installation of the application for you.
请注意， 如果你使用的是Android Studio， 那你不直接用adb (或 aapt)在实例上安装应用。 安卓 Studio会替你处理打包和安装

Forwarding Ports 转向端口
You can use the forward command to set up arbitrary port forwarding — forwarding of requests on a specific host port to a different port on an emulator/device instance. Here's how you would set up forwarding of host port 6100 to emulator/device port 7100:
你可以使用 forward命令来建立任意端口转向 - 将实例上指定的host端口转向另一个端口
adb forward tcp:6100 tcp:7100
You can also use adb to set up forwarding to named abstract UNIX domain sockets, as illustrated here:
你也可以使用adb来建立端口号到命名 抽象UNIX域名套接字命名的转向
adb forward tcp:6100 local:logd

Copying Files to or from an Emulator/Device Instance 复制或粘贴实例文件
You can use the adb commands pull and push to copy files to and from an emulator/device instance. Unlike theinstall command, which only copies an APK file to a specific location, the pull and push commands let you copy arbitrary directories and files to any location in an emulator/device instance.
可以使用pull和push命令来复制或粘贴实例文件， 不同于install命令 只能辅助APK文件到指定的位置， Push和pull命令可以允许你复制实例中的任意文件， 或者粘贴文件到任意位置

To copy a file or directory (and its sub-directories) from the emulator or device, use从实例中复制文件或文件夹

adb pull <remote> <local>
To copy a file or directory (and its sub-directories) to the emulator or device, use 粘贴文件或文件夹到实例中
adb push <local> <remote>
In the commands, <local> and <remote> refer to the paths to the target files/directory on your development machine (local) and on the emulator/device instance (remote). For example:
命令中的<local> 和 <remote>指的是开发机器上的路径和实例上的路径
adb push foo.txt /sdcard/foo.txt

Stopping the adb Server 停止adb服务

In some cases, you might need to terminate the adb server process and then restart it to resolve the problem (e.g., if adb does not respond to a command).

To stop the adb server, use the kill-server command. You can then restart the server by issuing any other adb command.
在有些情况下你可能需要终止adb服务然后重启来解决某些问题（例如 adb没有响应某个命令）。 停止服务使用 kill-server 命令。你可以通过使用其他任何adb命令重启服务

Wireless usage无线使用
adb is usually used over USB. However, it is also possible to use over Wi-Fi, as described here.
adb通常需要USB连接。 不过， 你也可以通过WIFI 连接
  1. Connect your Android device and adb host computer to a common Wi-Fi network accessible to both. We have found that not all access points are suitable; you may need to use an access point whose firewall is configured properly to support adb.
    确保你的安卓设备WIFI与机器连在同一个可以访问的网络下。 我们发现不是所有的访问端口都适合， 你可能需要使用防火墙支持adb的访问端口
    Note: If you are attempting to connect to a Wear device, force it to connect to Wi-Fi by shutting off Bluetooth on the phone connected to it.
    注意： 如果你想试图连接到穿戴设备时， 必须要强制连接WIFI并关掉与设备的蓝牙连接

  2. Connect the device to the host computer with a USB cable. 
    用USB数据线连接设备到Host机器上
  3. Set the target device to listen for a TCP/IP connection on port 5555.启用5555 端口号让设备来监听TCP/IP 连接
  4. $ adb tcpip 5555
  5. Disconnect the USB cable from the target device. 移除数据线
  6. Find the IP address of the Android device. For example, on a Nexus device, you can find the IP address at Settings >About tablet (or About phone) > Status > IP address. Or, on an Android Wear device, you can find the IP address atSettings > Wi-Fi Settings > Advanced > IP address.
    找到安卓设备的IP地址。 Nexus设备的IP地址在设置>关于手机或平板>状态>IP 地址； 或者也可以在设置>WIFI 设置>高级>IP地址中查询
  7. Connect to the device, identifying it by IP address.
    通过IP地址指向设备，　连接设备
    $ adb connect <device-ip-address>
  8. Confirm that your host computer is connected to the target device:
    确认你的机器连接到了设备
    $ adb devices
List of devices attached
<device-ip-address>:5555 device
You're now good to go!现在可以了
If the adb connection is ever lost:　如果adb连接中途丢失了
  1. Make sure that your host is still connected to the same Wi-Fi network your Android device is. 确保你的机器鱼设备WIFI是连在同一网络下
  2. Reconnect by executing the "adb connect" step again.通过 adb connect命令步骤 重新连接
  3. Or if that doesn't work, reset your adb host:或者如果不工作的话， 那重设host 
    adb kill-server
    and then start over from the beginning.然后从头开始
